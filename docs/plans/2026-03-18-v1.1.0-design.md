# FootNote Writer v1.1.0 Design

## Overview

v1.1.0 包含 4 项改动：修复 Windows DOCX 脚注 bug、支持多 LLM 服务商、脚注/尾注双模式导出、更多引注格式预设。

---

## 1. Windows DOCX 脚注 Bug 修复

**症状**：Windows 上导出的 DOCX 脚注未转换为原生 Word 脚注，正文残留 `[^footnote0]` 纯文本。Mac 上同样内容正常。

**修复策略**（防御性修复 + 日志增强）：

- **扩展脚注解析器**：`parseFootnotes()` 增加对 `[^footnoteN]`、`[^fn_N]`、`[^fn-N]` 等非标准格式的支持
- **DOCX 导出诊断日志**：导出前记录脚注 ID 映射和段落内容到 Electron 日志
- **导出诊断按钮**：UI 增加一个可选的「复制调试信息」功能，方便用户反馈

### 涉及文件
- `src/renderer/lib/footnote-parser.ts` — 扩展正则
- `src/main/docx-export.ts` — 增加日志
- `src/renderer/App.tsx` — 可选的诊断按钮

## 2. 多 LLM 服务商支持

```typescript

interface LLMProvider {
  id: string                              // 'minimax' | 'deepseek' | 'kimi' | 'qwen' | 'openai' | 'custom'

```

## 3. 脚注/尾注双模式导出

### 三种导出模式

| 模式 | 正文标记 | 注释位置 | 实现方式 |
|------|---------|---------|---------|
| 脚注（默认） | Word 原生脚注上标 | 每页页脚 | 当前实现，不变 |
| 尾注 | 上标数字 ¹²³ | 文末「注释」段落列表 | 文末段落模拟 |
| 参考文献列表 | 上标 [1][2] | 文末「参考文献」段落列表 | 文末段落模拟 |

### UI
- 导出按钮旁增加下拉选择导出模式
- 默认为「脚注」

### 技术方案
- `exportToDocx()` 新增 `mode: 'footnote' | 'endnote' | 'bibliography'` 参数
- 尾注/参考文献模式：用 `SuperscriptRun` + 文末 `Paragraph` 列表实现
- 复用现有的脚注解析结果

### 涉及文件
- `src/main/docx-export.ts` — 新增尾注/参考文献导出逻辑
- `src/renderer/App.tsx` — 导出模式选择 UI
- `src/shared/types.ts` — ExportOptions 增加 mode 字段

## 4. 更多引注格式预设

### 模板分组

**法学引注手册（保留现有 7 个）**

**GB/T 7714（新增 4 个）**：
- 专著：`[序号] 作者.书名[M].出版地:出版者,出版年:页码.`
- 期刊：`[序号] 作者.文章题名[J].期刊名,年,卷(期):页码.`
- 学位论文：`[序号] 作者.题名[D].保存地点:保存单位,年份.`
- 网络文献：`[序号] 作者.题名[EB/OL].(发表日期)[引用日期].URL.`

**APA 第 7 版（新增 2 个）**：
- 书籍：`Author, A. A. (Year). *Title of work*. Publisher.`
- 期刊：`Author, A. A. (Year). Title of article. *Journal Name*, *Volume*(Issue), Pages.`

**Chicago（新增 2 个）**：
- 脚注式：`Author, *Title* (Place: Publisher, Year), Page.`
- 书目式：`Author. *Title*. Place: Publisher, Year.`

### 数据结构改动

```typescript
interface FootnoteTemplate {
  id: number
  name: string
  format: string
  isPreset: boolean
  group: string | null  // 新增：模板分组名
}
```

数据库 `footnote_templates` 表新增 `group_name TEXT` 列。

### System Prompt 联动
- 选择不同格式组时，system prompt 中的「脚注格式」说明自动替换为对应规范
- 每个格式组定义一段完整的格式说明文本

### UI 改动
- `FootnoteFormatEditor` 按 group 分组显示模板
- 用户自定义模板归入「自定义」分组

### 涉及文件
- `src/main/db.ts` — 数据库迁移 + 新预设种子数据
- `src/shared/types.ts` — FootnoteTemplate 增加 group 字段
- `src/renderer/components/FootnoteFormatEditor.tsx` — 分组显示
- `src/main/ipc.ts` — system prompt 格式说明联动
